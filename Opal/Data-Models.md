## Session Layer

### SessionSnapshot

The primary document stored in MongoDB per session.

| Field | Type | Description |
|-------|------|-------------|
| `session_id` | `string` | UUID |
| `lifecycle_stage` | `'moodboard' \| 'planning' \| 'celebrate' \| 'memory'` | Current stage |
| `plan_ready` | `bool` | Can the Plan tab be activated |
| `moodboard` | `MoodboardItem[]` | Visual artifacts |
| `text_cards` | `BoardTextCard[]` | Tip/insight cards |
| `next_questions` | `string[]` | Latest AI follow-up questions (56-candidate pool) |
| `poster_url` | `string \| null` | Auto-generated 1600×1200 WEBP collage |
| `poster_version` | `string \| null` | Hash — use as `<img key>` |
| `chat_turns` | `ChatTurn[]` | Full conversation history |
| `entities` | `dict` | Extracted event entities (occasion, guest_count, budget, venue, theme…) |
| `party_plan` | `PartyPlan \| null` | Structured plan with sections |
| `public_slug` | `string` | URL slug for sharing |
| `is_public` | `bool` | Published state |
| `view_count` | `int` | Deduplicated views |
| `share_card_url` | `string \| null` | OG card 1200×630 |
| `fork_count` | `int` | Times forked |
| `parent_session_id` | `string \| null` | Fork lineage |
| `version_label` | `string` | `'v1'` → `'v2'` per fork |

### MoodboardItem

22 fields. Key groups:

**Identity:** `artifact_id` (guaranteed non-empty), `label`, `category`, `image_source`

**URLs:** `image_url` (active display), `r2_url` (permanent), `cdn_url` (short TTL)

**Interaction signals (feed AI feedback loop):**

| Field | Type | Effect on AI |
|-------|------|-------------|
| `thumbs` | `-1 \| 0 \| 1` | +1 steers questions toward this aesthetic; -1 avoids it |
| `is_pinned` | `bool` | Style anchor — highest signal |
| `regen_count` | `int` | Implicit negative signal |
| `downloaded` | `bool` | Engagement signal |
| `shared` | `bool` | Engagement signal |
| `saved` | `bool` | Engagement signal |

**Provenance:** `generation_model`, `generation_provider`, `enrichment_latency_ms`, `position_in_turn`, `confidence`

### ChatTurn

| Field | Type | Description |
|-------|------|-------------|
| `turn_number` | `int` | Sequential |
| `user_message` | `string` | User input |
| `assistant_message` | `string` | Full LLM response |
| `artifact_ids` | `string[]` | Links to MoodboardItem.artifact_id |
| `text_card_ids` | `string[]` | Links to BoardTextCard.card_id |
| `created_at` | `string` | ISO-8601 |

## Taste Profile Layer

### UserTasteProfile

| Field | Type | Description |
|-------|------|-------------|
| `profile_version` | `int` | Use as SWR cache key |
| `total_events_planned` | `int` | Finalized session count |
| `global_profile` | `TasteSubProfile` | Aggregated across all event types |
| `event_type_profiles` | `dict[str, TasteSubProfile]` | Per-event-type breakdown |
| `budget_pattern` | `BudgetPattern` | avg, min, max, distribution |
| `snapshots` | `list` | Per-session extraction history (max 20) |

Auto-populated when a session is finalized. Injected into Kyra's system prompt at turn 0 for returning users.

## Calendar Layer

### CelebrationDate

| Field | Type | Description |
|-------|------|-------------|
| `label` | `string` | e.g. "Mum's 60th" |
| `event_type` | `string` | `birthday \| anniversary \| wedding \| graduation \| other` |
| `month` / `day` | `int` | Date of occurrence |
| `year` | `int \| null` | null = recurring annually |
| `is_recurring` | `bool` | Auto-rolls past dates to next year |
| `next_occurrence` | `string` | YYYY-MM-DD — calculated by BE, never client-side |
| `source` | `string` | `'manual' \| 'blueprint_extract'` |
| `proactive_dismissed` | `bool` | User dismissed proactive suggestion for this date |

### DraftBlueprint (Proactive)

Auto-generated by scheduler for CelebrationDates within 90 days.

| Field | Type | Description |
|-------|------|-------------|
| `is_draft` | `bool` | `true` until user approves |
| `is_proactive` | `bool` | Always `true` for scheduler-generated drafts |
| `completion_score` | `float` | 0–100 |
| `celebration_date_id` | `string` | Links to the triggering CelebrationDate |

## Recap Layer

### BlueprintRecap

Submitted after the event happens. Transitions session to `'memory'` stage.

| Field | Type | Description |
|-------|------|-------------|
| `rating` | `int (1–5)` | Overall event rating |
| `notes` | `string \| null` | Free-text reflection |
| `photo_urls` | `string[]` | CDN URLs (max 10) |
| `actual_budget` | `float \| null` | Actual spend |
| `vendor_ratings` | `list` | `[{vendor_category, vendor_name?, rating, note?}]` |

## Intelligence Layer

### AnalyzerResult

Output of the LLM analyzer (or deterministic fast-path).

| Field | Type | Description |
|-------|------|-------------|
| `intent_modes` | `list[IntentMode]` | Primary + secondary intents |
| `injection_detected` | `bool` | Prompt injection flag — triggers fast-exit |
| `safety_flags` | `list[str]` | `['none']` or `['prompt_injection']` |
| `entities` | `dict` | Extracted event facts |
| `missing_info_questions` | `list[str]` | What the AI still needs to know |
| `user_intent` | `string` | Derived intent description |
| `tone_style` | `string` | `'collaborative'` etc. |
| `complexity_level` | `string` | `'low' \| 'medium' \| 'high'` |
| `shadow_plan_stage` | `ShadowPlanStage` | Plan lifecycle |

### IntentMode (12 modes)

`event_planning` · `theme_curation` · `drinks_bar` · `venue_discovery` · `attire_advice` · `quick_tips` · `decision_advisory` · `inspiration` · `budget_planning` · `vendor_sourcing` · `recap_review` · `hosting_help`

### WorkItemState (session-scoped)

Tracks active planning work item decisions across turns.

Key `decisions` keys:
- `questions_asked: list[str]` — cross-turn dedup (never re-ask)
- `pinned_artifacts: list[str]` — steers question generation
- `rejected_artifacts: list[str]` — avoided in future questions
- `sticky_facts: dict` — confirmed entities that survive context truncation
